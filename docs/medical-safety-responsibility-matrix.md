# 医療情報安全管理 責任分界表

更新日: 2026-04-19

## 目的

医療情報システム安全管理ガイドラインで求められる `責任分界` を、このプロジェクトの現行構成に合わせて可視化する。

## 前提

- 本システムは `EMS / HOSPITAL / ADMIN / DISPATCH` が利用する救急搬送支援システムである。
- 現時点の repo には運用文書はあるが、通常時 / 非常時 / 委託時の責任分界を 1 枚で確認できる文書がなかった。
- ここでいう `開発保守担当` は本アプリの開発・保守を行うチーム、`クラウド / 外部事業者` は hosting、DB、監視、backup などの提供者を含む。

## 役割定義

| 役割 | このプロジェクトでの想定 |
|---|---|
| 医療機関等の管理責任者 | 導入組織側の最終責任者。運用承認、委託承認、監査指示、非常時判断を担う |
| 企画管理者 | 導入・運用設計の責任者。規程整備、リスク評価、権限方針、教育計画を担う |
| システム運用担当者 | `ADMIN` を中心とした日次運用担当。監視、アカウント、端末、点検、一次切り分けを担う |
| 現場利用部門責任者 | `EMS` / `HOSPITAL` / `DISPATCH` の各現場責任者。利用者管理依頼、教育実施、現場逸脱報告を担う |
| 開発保守担当 | アプリ改修、障害解析、ログ確認、セキュリティ対応、リリースを担う |
| クラウド / 外部事業者 | インフラ、DB、監視、backup、通信などの提供者。契約範囲内の保守・報告を担う |

## 委託 / クラウド利用時の原則

- 医療機関等の管理責任は委託で移転しない。
- 契約、SLA、障害報告、再委託、ログ提供、backup 責任、restore 支援、監査協力を文書で明示する。
- 外部事業者が対応する項目でも、導入組織側に `確認責任` と `承認責任` が残る。

## 実名運用の固定ルール

- 導入組織ごとの実名責任者、代行者、緊急連絡順、証跡保管場所は [medical-safety-responsibility-assignment-template.md](/C:/practice/medical-support-apps/docs/reference/medical-safety-responsibility-assignment-template.md) を使って別紙化する。
- 実名版の保管先は `主保管場所` と `副保管場所` を 1 つずつ固定し、月次点検で参照可能性を確認する。
- 監査、ID 棚卸、本人確認、BCP、教育、委託先資料の保管責任者は、実名版の `記録保管場所` 表で必ず実名にひも付ける。
- repo には様式だけを置き、実名版は導入組織の管理統制下にある保存先で管理する。

## 現時点の不足

- 導入組織ごとの実名アサイン欄と保管場所の `様式` は追加したが、実データ投入は未了。
- クラウド / 監視 / backup / 通知の各事業者を分けた責任表がまだない。
- 通常時と非常時で参照する契約書、SLA、連絡先一覧の実保管先は導入組織ごとに未固定。
- 地域連携や第三者提供が入る場合の追加責任分界は別紙が必要。

## 次にやること

1. 導入組織ごとの実名版を別紙で実データ入力する
2. 監視、backup、hosting、DB の外部事業者を個別に棚卸する
3. 本表に対応する契約 / runbook / 連絡先一覧を `規程 / 証跡一覧` にひも付ける
