# 搬送調整支援システム 病院向け説明文書

更新日: 2026-04-19

## 目的

この文書は、救急搬送時の病院調整・受入調整を支援する本システムについて、病院側の導入説明、運用説明、責任分界確認のために、目的と情報の扱いを簡潔に説明するための文書である。

## この文書の位置付け

- この文書は `導入先別の対外説明文書` の最初のひな形として扱う。
- 現在は `病院向け` を正本にしているが、今後 `EMS 向け`、`自治体 / 本部向け`、`連携先向け` などを同じ構成で追加してよい。
- 追加時は、共通部分を流用しつつ、対象ごとの `利用目的`、`閲覧範囲`、`運用条件`、`問い合わせ経路` だけを差し替える。

## 今後の追加ルール

- 導入先別の文書は `docs/policies/external-explanation-for-transport-coordination-system-<target>.md` 形式で追加する。
- `<target>` には `hospital`、`ems`、`dispatch`、`municipality`、`partner` などの導入先種別を入れる。
- 新しい導入先文書を追加したら、`導入時に説明する項目`、`問い合わせ窓口`、`外部保存の説明` の 3 点が欠けていないか確認する。
- 導入組織固有の実名、窓口、説明日、承認者は、文書本文ではなく導入記録または責任者割当実名版で管理する。

## システムの位置付け

- 本システムは、`救急搬送時の病院調整・受入調整を支援するシステム` である。
- 現時点では、`正式な電子カルテ本体ではない`。
- 将来、診療録等や電子カルテとの連携を検討する可能性はあるが、現時点では別システムとして扱う。

## 主な利用目的

- 救急隊による搬送先調整
- 病院による受入可否の確認と相談
- 搬送判断の共有
- 関係者間の状況確認
- 運用監査と障害対応

## 取り扱う情報の範囲

- 搬送調整に必要な患者情報
- 受入要請履歴
- 相談履歴
- 搬送判断に関する情報
- 監査および運用管理に必要な情報

## 正式記録としての扱い

- 本システムは現時点で `搬送調整支援システム` であり、`正式な診療録等の保存主体` ではない。
- 病院内で法令や院内規程に基づいて保存すべき正式記録は、病院側の正本システムまたは定められた記録媒体で管理する。
- 本システム上の情報は、搬送調整、受入判断、相談経過、運用監査のために利用する。

## アクセスできる者

- `EMS`
- `HOSPITAL`
- `ADMIN`
- `DISPATCH`

上記の各利用者は、それぞれの役割と必要範囲に応じて、アクセス可能な情報を制限される。

## 第三者提供と外部保存の考え方

- 本システムでは、搬送調整支援に必要な範囲で、導入組織が管理する外部基盤や委託先基盤を利用することがある。
- 外部基盤を利用する場合でも、医療機関等の責任の下で、契約、責任分界、障害報告、証跡確認を行う。
- 本システムの利用に伴って、病院利用者が任意の外部サービスへ患者情報を持ち出してよいことにはならない。
- 将来の地域連携、外部 API、電子カルテ連携は、別途の責任分界と合意が整うまで前提にしない。

## 病院側にお願いする運用条件

- 利用端末は `院内の公用端末` に限定する
- 接続は `院内LAN` からのみ行う
- `ゲストWi-Fi`、`管理外ネットワーク`、`持込端末` からの利用は行わない
- 同一拠点内であれば、複数の公用端末を登録して運用してよい
- 拠点外からの例外アクセスは行わない
- 端末故障時は、別の公用端末へ切り替え、必要な権限移管を行う
- 障害時にシステム利用が困難な場合は、電話等の代替手段へ切り替える

## 安全管理の考え方

- 利用者認証と権限制御を行う
- 公用端末を前提に運用する
- 監査ログおよび監視記録を保持する
- バックアップおよび復旧手順を整備する
- 委託先や外部基盤を利用する場合でも、責任分界と安全管理を明確にする

## 外部事業者 / クラウド基盤の利用

- 本システムでは、安全管理措置を講じた外部事業者やクラウド基盤を利用する場合がある。
- その場合も、医療機関等の責任の下で、契約、管理、監督を行う。
- 事業者名、契約番号、直接連絡先などの詳細は内部管理資料で管理し、必要な範囲だけ導入組織へ説明する。
- 外部保存や再委託の有無、保存場所、障害報告経路、削除 / 契約終了時の扱いは内部台帳で確認する。

### 主な委託カテゴリ

- アプリ実行基盤
- データベース基盤
- バックアップ保存先
- 監視 / 障害検知基盤
- 通知連携基盤

各カテゴリについては、役割と責任分界を内部台帳で管理する。

## 外部保存についての説明

- データの一部は、契約と安全管理の下で外部基盤に保存される場合がある。
- 保存先の選定にあたっては、責任分界、ログ提供、障害報告、再委託、削除証跡の観点から確認を行う。
- 病院側には、導入時に `どのカテゴリの外部基盤を利用するか`、`問い合わせ先はどこか`、`障害時にどう切り替えるか` を説明する。

## 問い合わせと障害時

- 利用上の問い合わせや障害時は、導入時に定めた `所属責任者`、`運用窓口`、`緊急連絡先` の順で連絡する。
- ログイン不能、受入要請が見えない、相談送信不可、通知停止、端末紛失などは、問い合わせ時にロール、端末名、発生時刻、対象案件 ID を添えて連絡する。
- 障害時には、必要に応じて電話等の代替手段へ切り替える。
- サイバー攻撃や不正アクセス疑いがある場合は、通常問い合わせではなく緊急連絡ルートを使う。

## 病院側で定めていただく窓口

- 管理責任者
- 企画管理者
- システム運用担当
- 拠点ネットワーク担当
- 現場責任者

上記の実名、代行者、連絡順、記録保管場所は導入時に別紙で確定する。

## 病院側で事前に確認いただきたいこと

- 院内LANで利用できる公用端末が用意されていること
- 利用者と端末の管理責任者が決まっていること
- 障害時の連絡窓口が決まっていること
- 院内での代替連絡手段が決まっていること
- 外部保存や委託先利用について、院内の説明先が決まっていること
- 正式記録の保存先と、本システムの利用目的の違いが院内で共有されていること

## 導入時に説明する項目

- 本システムの位置付けと正式記録との違い
- 利用できるロールと閲覧範囲
- 公用端末、院内 LAN、問い合わせ経路などの利用条件
- 外部基盤を利用する場合のカテゴリ、責任分界、障害時の案内
- 代替連絡手段と、利用停止時の切替方法

## 仮データ例

- 導入先名: `SAMPLE 総合病院`
- 説明対象: `病院管理者 / 地域連携担当 / 救急外来責任者`
- 通常問い合わせ窓口: `sample-hospital-support@example.jp`
- 緊急連絡先: `03-0000-0001`
- 代替連絡手段: `電話連絡 / FAX / 院内既存連絡網`
- 説明日: `2026-05-01`
- 承認者: `SAMPLE 企画管理者`

## 他の導入先へ展開するときに差し替える項目

- 利用者の想定ロール
- 閲覧 / 操作できる情報の範囲
- 接続元と端末条件
- 問い合わせ窓口と障害時の連絡順
- 導入先が事前に決める責任者
