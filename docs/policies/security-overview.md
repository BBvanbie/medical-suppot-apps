# セキュリティ方針概要

## 目的

- 外部説明用に「このシステムで何をどう守っているか」を短く示す。

## 現在の保護方針

- ロールごとの画面 / API 制御
- ログイン失敗回数制限と一時ロック
- 端末ごとの PIN 再入場
- セッション失効制御
- 一時パスワードと強制変更
- 端末登録コードによる登録済み端末運用
- 監査ログ記録
- 監視イベント記録

## 役割ごとの考え方

- `EMS`
  - 自隊データだけを扱う
- `HOSPITAL`
  - 自院宛要請だけを扱う
- `ADMIN`
  - 管理と監視を担う
- `DISPATCH`
  - 指令系機能を担う

## 端末保護

- `EMS / HOSPITAL` は登録済み端末前提で運用する
- 3時間無操作で PIN 再入場
- 8時間で完全再ログイン
- 紛失時はアカウント停止を正本にする

## 運用保全

- 1日2回バックアップ
- 14日保持
- `ADMIN` 監視画面で主要異常を確認

## 今後の拡張余地

- 端末 fingerprint 強化
- `ADMIN / DISPATCH` の MFA 必須化
- backup run の自動報告
