# セキュリティ方針概要

## 目的

- 外部説明用に「このシステムで何をどう守っているか」を短く示す。

## 現在の保護方針

- ロールごとの画面 / API 制御
- ログイン失敗回数制限と一時ロック
- `EMS / HOSPITAL` の WebAuthn MFA
- 登録済み端末運用
- セッション失効制御
- 一時パスワードと強制変更
- 端末登録コードによる登録済み端末運用
- 監査ログ記録
- 監視イベント記録
- オフライン保存データの TTL とログアウト時削除

## 役割ごとの考え方

- `EMS`
  - 自隊データだけを扱う
- `HOSPITAL`
  - 自院宛要請だけを扱う
- `ADMIN`
  - 管理と監視を担う
- `DISPATCH`
  - 指令系機能を担う

## 端末保護

- `EMS / HOSPITAL` は登録済み端末前提で運用する
- `EMS / HOSPITAL` はログアウト後のログインで WebAuthn MFA を必須にする
- セッションは 5 時間で完全再ログインを求める
- 現段階では PIN 再入場はログイン導線に組み込まない
- 紛失時はアカウント停止を正本にする
- ログアウト時は IndexedDB の保護対象データを削除する
- 起動時に TTL を超えたオフライン保存データを削除する

## 運用保全

- 1日2回バックアップ
- 14日保持
- `ADMIN` 監視画面で主要異常を確認

## 今後の拡張余地

- 端末 fingerprint 強化
- `ADMIN / DISPATCH` の MFA 必須化
- backup run の自動報告
- IndexedDB の Web Crypto 暗号化
