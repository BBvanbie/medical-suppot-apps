# インフラ構成説明

## 目的

- 導入先や関係者へ説明できる最小構成資料を用意する。

## 構成要素

- フロントエンド / サーバー:
  - Next.js App Router
- アプリ実行基盤:
  - Node.js ランタイム
- DB:
  - PostgreSQL
- 認証:
  - NextAuth Credentials ベース

## 主なデータ経路

1. 利用者がブラウザからログインする
2. アプリが認証とロール判定を行う
3. PostgreSQL に事案、要請、通知、監査、認証補助データを保存する

## 主な保存データ

- 事案
- 受入要請
- 要請 target
- 相談履歴
- 患者一覧
- 通知
- 監査ログ
- 認証補助データ

## 運用補助

- バックアップ:
  - 12:00 / 24:00
- 監視:
  - `ADMIN / 監視`
- 端末運用:
  - 登録済み端末ベース

## 補足

- 現時点では外部監視 SaaS や IdP を前提にしない
- まずはアプリ内監視と DB ベースの運用を正本にする
