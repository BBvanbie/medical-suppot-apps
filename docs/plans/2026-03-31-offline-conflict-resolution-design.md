# オフライン競合解決仕様設計

## 結論

- オフライン競合は `手動解決` を基本方針とし、自動 merge は行わない。
- 競合検知時は `server優先` で自動同期を停止し、EMS 利用者へ「フォームで確認して保存し直す」導線を出す。
- 競合対象は未送信キューと事案編集画面の両方で見えるようにする。

## 今回の実装対象

- 競合ルールの docs 明文化
- 未送信キューでの競合項目の意味付け整理
- 事案編集画面で競合復元時の最小表示
- 競合復元 E2E の追加

## 競合ルール

- `case_update`
  - ルール: `手動解決`
  - 方針: `server優先`
  - UI: フォーム復元時に競合注意を表示し、利用者が内容を確認して再保存する
- `settings_sync`
  - ルール: `server優先`
  - 方針: 自動同期は停止し、未送信キューで確認
- `hospital_request_send` / `consult_reply`
  - 競合よりも送信可否失敗扱いを優先し、未送信キューで手動再送する

## 非目標

- サーバー値とローカル値の差分比較 UI
- 自動マージ
- HOSPITAL / ADMIN 側の競合解決 UI
