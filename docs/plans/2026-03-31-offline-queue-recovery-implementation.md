# 未送信キュー回復導線実装手順

## 目的

- 未送信キューを「溜めるだけ」の画面から「分類して回復する」画面へ更新する。

## 変更内容

1. `OfflineQueueItem` に失敗理由分類と推奨アクションを追加する。
2. 同期失敗時に `failureKind` / `recoveryAction` を埋める。
3. 未送信キュー画面に一括再送と理由表示を追加する。
4. E2E を更新し、再送成功と失敗分類を確認する。

## 実装順

1. 型拡張
2. 失敗分類ヘルパー追加
3. 同期 / 手動送信への分類適用
4. キュー UI 更新
5. テスト更新

## 運用方法

- 送信失敗時は通信状態バナーではなく未送信キュー画面で内容確認する。
- `retry` の項目は再送、`review` は内容確認、`discard` は不要時のみ破棄する。

## 注意点

- 既存の日本語 UI 文言を壊さない。
- `case_update` と `settings_sync` の自動同期挙動は維持する。
- 競合は `retry` ではなく `review` に分類する。
