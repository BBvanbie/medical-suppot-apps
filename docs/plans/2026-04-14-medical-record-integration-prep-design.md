# 診療録等 / 電子カルテ連携 preparation design

作成日: 2026-04-14

## 結論

- 当面の主体は `搬送調整支援システム` とする。
- そのうえで、後続フェーズとして `診療録等連携`、`電子カルテ連携` を別テーマで準備する。

## 連携を後段化する理由

- 現行システムには電子署名、確定操作、代行入力承認、長期保存検証の前提がない。
- まず搬送調整そのものの安全管理、監査、運用証跡、BCP を固める方がリスクが低い。
- 連携要件は、相手先の電子カルテ、標準形式、責任分界、データ境界に大きく依存する。

## 後続で整理する論点

1. どのデータを連携対象にするか
   - 事案概要
   - 受入要請履歴
   - 相談履歴
   - 搬送判断
   - 到着後診療情報
2. どちらを正本にするか
   - 本システム正本
   - 電子カルテ正本
   - 項目ごとの二重管理禁止
3. 連携方式
   - CSV / PDF 出力
   - API 連携
   - HL7 FHIR 等の標準化
4. 責任分界
   - 連携失敗時
   - 反映遅延時
   - 監査 / 証跡の所在
5. 正式記録化が必要になった場合の追加要件
   - 確定操作
   - 電子署名 / タイムスタンプ
   - 代行入力承認

## 現時点の次アクション

- 今は実装しない。
- `搬送調整システム` の安全管理 docs を固めた後、連携対象データ一覧を別紙化する。
