# EMS 患者情報 OCR 保険証拡張 設計

最終更新: 2026-04-22

## 結論

- 既存の EMS 患者情報 OCR に `健康保険証` と `資格確認書` を追加する。
- 抽出対象は既存どおり `氏名 / 住所 / 生年月日` に限定し、保険者番号や記号番号などの保険情報は保存しない。
- Python 側は `insurance_card` と `eligibility_certificate` を同じ抽出ルール群で扱う。

## 前提と制約

- 既存 UI / API / Python OCR の構成は維持する。
- 保険証系は券面差分が大きいため、初期版は label-based 抽出を優先し、保険情報の詳細抽出には入らない。
- 住所は券面に存在しない場合があるため、未抽出は許容する。

## 方式

### 1. UI

- `本人確認書類を読み取る` の種別選択に `健康保険証` と `資格確認書` を追加する。

### 2. API

- `POST /api/ems/patient-identity-ocr` の許可種別に `insurance_card` と `eligibility_certificate` を追加する。

### 3. Python OCR

- 新しい書類種別は共通の `保険証系抽出` へ寄せる。
- 氏名は `氏名 / 被保険者氏名 / 被保険者名 / 受給者氏名` を候補にする。
- 住所は `住所 / 被保険者住所 / 住所又は居所` を候補にする。
- 住所正規化では `保険者 / 記号 / 番号 / 枝番 / 負担割合 / 資格取得 / 有効` 以降を切る。
- 公開サンプルで確認できた `健康保険資格確認書 / 資格情報のお知らせ / 被保険者証` のラベル差分を優先して取り込む。

## 公開サンプル参照

- 厚生労働省の `資格確認書等に関する運用状況調査結果` は運用文書中心で、券面ラベル確認には弱い。
- 協会けんぽ大阪の 2026年4月号 PDF に `健康保険資格確認書` と `資格情報のお知らせ` の見本があり、以下のラベルを確認できた。
  - `健康保険資格確認書`
  - `被保険者（本人）`
  - `氏名`
  - `生年月日`
  - `資格取得年月日`
  - `有効期限`
  - `保険者番号 / 保険者名称 / 保険者所在地`
- これに合わせ、保険証系 OCR は `氏名 / 住所 / 生年月日` のみ返しつつ、保険情報側ラベルは stop word として扱う。

## 非目標

- 保険者番号、記号番号、枝番、負担割合の保存
- OCR 結果の完全自動反映
- 保険証系の券面個別最適化
