# EMS/病院連携システム 改修仕様書 (2026-03-06)

## 1. 目的
- A側(救急隊)とHP側(病院)の受入要請、相談、搬送決定/辞退の運用を統一する。
- 通知バッジとポップアップ通知を導入し、未読イベントをリアルタイムに近い形で可視化する。
- DB入出力(API送信/受信)の整合性を明確化し、status遷移の矛盾を防ぐ。

## 2. 共通UI
- 全ページ共通でトップバーとサイドバーを表示。
- A側メインカラー: 青。
- HP側メインカラー: 緑。
- status色はA/HPで共通:
  - 未読(UNREAD): 緑
  - 既読(READ): グレー
  - 受入可能(ACCEPTABLE): 青
  - 受入不可(NOT_ACCEPTABLE): 赤
  - 要相談(NEGOTIATING): オレンジ

## 3. A側(救急隊)

### 3.1 ホーム
- システム名: 救急搬送支援システム。
- サイドバー: ホーム / 事案作成 / 事案一覧 / 病院検索 / 設定。
- サイドバー下部にログインIDに紐づく救急隊名を表示。
- ハンバーガーで開閉可能。
- メインは各ページ遷移カードを表示。

### 3.2 事案一覧ページ
- 親行(事案):
  - `incidentId / 覚知日時 / 指令先住所 / 氏名 / 年齢 / 性別 / status / 搬送先 / 詳細`
- 親行クリックで展開/収納。
- 子行(病院依頼):
  - `病院名 / 科目 / 送信日時 / status / 最新病院コメント / 最新救急隊コメント / 搬送決定 / 搬送辞退 / 相談`
- 子行ボタン押下は親行の展開トグルを発生させない。

#### ボタン有効条件
- 搬送決定: `requestStatus == ACCEPTABLE` のときのみ有効(青)。
- 搬送辞退: 常に有効(赤)。
- 相談: `requestStatus == NEGOTIATING` のときのみ有効(オレンジ)。

#### 相談ボタン
- 画面遷移せず、共通相談チャットモーダルを表示。
- 同モーダルは事案詳細の選定履歴/相談一覧からも共通利用する。

### 3.3 事案詳細ページ
- タブ構成: `基本情報/概要 / バイタル / 患者サマリー / 病院検索 / 選定履歴 / 相談一覧`。
- アクティブタブは青の角丸矩形で表示し、切替時に移動アニメーション。
- 相談一覧:
  - 病院検索で送信済みのうち `NEGOTIATING` の病院を一覧表示。
  - 行構造は事案一覧の子行と同等。
  - 相談ボタンで共通相談チャットモーダルを表示。
- 選定履歴:
  - 送信した全病院を一覧表示(相談一覧と同形式)。

## 4. HP側(病院)

### 4.1 サイドバー
- `ホーム / 受入要請一覧 / 搬送患者一覧 / 相談事案一覧 / 搬送辞退患者一覧 / 診療情報入力 / 設定`

### 4.2 受入要請一覧
- テーブル: `事案ID / A隊名 / 覚知日時 / 指令先住所 / 氏名 / 年齢 / 性別 / 選定科目 / status / 詳細`。
- 詳細押下: 患者サマリーモーダル表示。
- モーダル下部に `受入可能 / 受入不可 / 要相談` を配置。

### 4.3 搬送患者一覧
- 条件: 自院宛で `TRANSPORT_DECIDED` の事案。
- テーブル: `事案ID / A隊名 / 覚知日時 / 指令先住所 / 氏名 / 年齢 / 性別 / 選定科目 / 距離 / 詳細 / 科目修正`。
- 距離は指令先住所と自院の概算距離を表示。
- 詳細モーダルでは受入可能/要相談は非活性、受入不可のみ有効。

### 4.4 相談事案一覧
- 条件: `NEGOTIATING` 事案。
- テーブル: `事案ID / 救急隊名 / 覚知日時 / 指令先住所 / 氏名 / 年齢 / 性別 / 最新病院コメント / 最新救急隊コメント / 選定科目 / status / 詳細 / 相談`。
- 相談押下で患者サマリー+相談チャットを左右分割表示。
- すでに `TRANSPORT_DECIDED/TRANSPORT_DECLINED` の行はグレーアウトし操作不可。

### 4.5 搬送辞退患者一覧
- テーブル: `事案ID / A隊名 / 覚知日時 / 氏名 / 年齢 / 性別 / 選定科目 / 病院側回答 / 辞退受信日時`。

## 5. status送信ルール
- HP側から status 送信時は必ず確認モーダルを経由。
- 搬送決定後に受入不可へ変更する場合:
  - 送信確認後、警告モーダルで A隊名/電話番号を表示。
  - `電話連絡済み` 押下でクローズ可能。

## 6. 通知

### 6.1 通知バッジ
- ヘッダー右上にベルアイコン、未読件数を赤バッジ表示。
- メニュー/タブは件数ではなく赤ドットで未読を可視化。

### 6.2 ポップアップ通知
- 共通: 相談コメント受信時。
- A側: 病院status変化時(受入可能/受入不可/要相談)。
- HP側: 受入要請受信時、搬送決定通知受信時。

### 6.3 バッジ表示対象
- A側 status変更通知: ハンバーガー、事案一覧、事案詳細(選定履歴)。
- A側 相談コメント通知: ハンバーガー、事案一覧、事案詳細(相談一覧)。
- HP側 受入要請通知: ハンバーガー、要請一覧。
- HP側 搬送決定通知: ハンバーガー、搬送患者一覧。
- HP側 相談コメント通知: ハンバーガー、相談事案一覧、該当事案行。

## 7. DBデータモデル(送受信整合)
- `hospital_requests`: 事案単位の送信ヘッダ。
- `hospital_request_targets`: 病院ごとの現在status/選定科目/距離。
- `hospital_request_events`: status変更・相談コメントの履歴。
- `hospital_patients`: 搬送決定済み患者。
- `notifications`: 既読/未読を含む通知。

## 8. API整合
- A側送信: `POST /api/cases/send-history`。
- A側一覧取得: `GET /api/cases/search`。
- A側相談履歴/返信: `GET|PATCH /api/cases/consults/[targetId]`。
- A側搬送判断: `PATCH /api/cases/send-history`。
- HP側status更新: `PATCH /api/hospitals/requests/[targetId]/status`。
- 通知取得/既読化: `GET|PATCH /api/notifications`。

## 9. status遷移(要点)
- 初期: `UNREAD`。
- HP閲覧で `READ`。
- HP要相談送信で `NEGOTIATING`。
- HP受入可能で `ACCEPTABLE`。
- HP受入不可で `NOT_ACCEPTABLE`。
- A側判断で `TRANSPORT_DECIDED` または `TRANSPORT_DECLINED`。
- 遷移可否はサーバー側で検証し、不正遷移は409で拒否。

## 10. 今回の不具合修正内容
- `GET /api/cases/search` の SQLで `FROM cases` に別名がなく `c.*` 参照していた不整合を修正。
- APIの `limit` パラメータを数値検証付きで安全化。
- 失敗時メッセージを正常な日本語に修正。
- DB権限制約時に `ensure` 処理が落ち続けないよう、権限不足(42501)をフェイルソフト化。
