# 端末紛失時対応 Runbook

## 目的

- iPad、PC、予備端末の紛失や盗難が発生したときに、患者情報と業務導線を守りながら停止と再開を進める。

## 現在の基本方針

- 端末単独失効より先に `アカウント停止` を正本にする
- 既存セッションは失効させる
- 新端末または予備端末へ引継ぎ後に再開する
- 遠隔ワイプ機能は現時点では持たない
- ローカル保存の可能性があるため、紛失報告は即時扱いにする

## 対象端末の例

- `EMS` 用 iPad / タブレット
- `HOSPITAL` 用 PC
- 予備端末

## 初動

1. 利用者が所属責任者または `ADMIN` に連絡する
2. `ADMIN` が対象ユーザー、端末名、最終利用時刻を確認する
3. `ADMIN` がアカウント停止を行う
4. 既存セッション失効状態を確認する
5. 端末を紛失扱いとして記録する
6. 必要なら関係者へ一時利用停止を共有する

## 受付時に聞くこと

- 利用者名
- ロール
- 端末名
- 最後に使った日時
- 紛失場所の目安
- 端末にオフライン下書きが残っている可能性
- 予備端末の有無

## ADMIN の対応

### 停止

1. ユーザーを開く
2. アカウントを無効化する
3. 必要に応じて一時パスワードも無効とみなして再発行前提にする
4. 端末一覧で対象端末を紛失扱いにする

### 共有

- 所属責任者へ停止済みを伝える
- 必要なら当日の代替端末や代替運用を決める

## 再開手順

1. 新端末または予備端末を確定する
2. `ADMIN` が登録コードを発行する
3. 利用者が新端末でログインする
4. WebAuthn MFA を新端末で登録する
5. 端末登録コードを入力する
6. `ADMIN` がアカウントを再開する
7. 利用者が主要画面を確認する

## ローカルデータ方針

- 現状は remote wipe を持たない
- そのため、紛失端末には患者情報や下書きが残る可能性がある前提で扱う
- リスク低減は以下で行う
  - アカウント停止
  - セッション失効
  - 新端末への再登録
  - オフライン運用端末の棚卸し

## 再開後に確認すること

- ログインできる
- WebAuthn MFA が登録済みになっている
- 端末情報で登録済み端末になっている
- `EMS` ならホーム、事案作成、病院検索が開く
- `HOSPITAL` なら受入要請一覧、患者一覧が開く
- 通知が対象ロールへ戻っている
