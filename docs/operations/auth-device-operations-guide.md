# 認証 / 端末運用ガイド

## 目的

- `ID` と `username` の違い、端末登録から運用開始までの流れ、端末紛失時の新端末引継ぎを、誰が読んでも分かる形で整理する。
- `ADMIN`、導入担当、`EMS`、`HOSPITAL` の間で説明がぶれないようにする。

## まず用語を整理する

### 1. ID

- ログイン画面で入力する識別子。
- このプロジェクトでは、実際には `username` をログイン ID として使っている。
- 画面や現場説明では「ID」と言って問題ない。

例:

- `hospital_263`
- `e2e_hospital_a`
- `admin01`

### 2. username

- DB やシステム内部で使う正式なログイン名。
- 利用者が普段意識する「ログインID」と同じもの。

つまり:

- 現場説明の `ID`
- システム実装上の `username`

は、今の運用では同じものを指す。

### 3. display name

- 画面に表示される人や組織の見え方。
- ログインには使わない。

例:

- `災害医療センター`
- `本部機動第1`

### 4. パスワード

- `ID / username` と組み合わせてログインするときに使う秘密情報。
- 忘れた場合は電話連絡を受けた `ADMIN` が一時パスワードを発行する。

### 5. 登録コード

- 端末を正式端末として登録するときだけ使う一時コード。
- `ADMIN` が発行する。
- 一度使ったら再利用しない。

### 6. WebAuthn MFA

- EMS / HOSPITAL のログアウト後ログインで使う追加本人確認。
- iPad や PC の生体認証、端末 PIN、パスキーなどを使う。
- 端末登録とは別の確認であり、`deviceKey` の代替でもない。

### 7. PIN

- 現段階ではログイン導線に組み込まない。
- 将来補助的に使う場合でも、MFA の代替にはしない。

### 8. 端末名

- `ADMIN` が端末管理画面で見分けるための名前。

例:

- `EMS iPad`
- `病院受付PC`
- `災害医療センター 受付PC`

## このプロジェクトの現在の認証の考え方

- まず `ID / パスワード` で本人を確認する
- EMS / HOSPITAL は WebAuthn MFA で追加本人確認する
- 次に、その端末が正式端末かを確認する
- 5時間以上経ったら、もう一度 `ID / パスワード + WebAuthn MFA` からやり直す

対象:

- `EMS`: 端末登録あり
- `HOSPITAL`: 端末登録あり
- `ADMIN`: 現時点では端末登録なし
- `DISPATCH`: 現時点では端末登録なし

## 端末登録から運用開始までの流れ

### 全体像

1. `ADMIN` が利用者アカウントを用意する
2. `ADMIN` が一時パスワードを発行する
3. `ADMIN` が対象端末を用意する
4. `ADMIN` が登録コードを発行する
5. 利用者が端末でログインする
6. 未登録端末なら `端末登録画面` に進む
7. 利用者が登録コードを入力する
8. 登録完了後はいったん再ログインに戻る
9. ログアウト後のログインで WebAuthn MFA を通過する
10. `設定 > 端末情報` で `登録済み端末` と `WebAuthn MFA: 登録済み` を確認する

### ADMIN がやること

1. 利用者を作成する
2. 一時パスワードを発行する
3. `端末管理` で対象端末を確認する
4. 必要なら端末ロールと所属を設定する
5. `登録コード発行` を押す
6. コードを本人へ伝える

### EMS 利用者がやること

1. iPad でログイン画面を開く
2. `ID / パスワード` を入力する
3. `端末登録` 画面が出たら、登録コードを入力する
4. 端末登録完了後に再ログインする
5. WebAuthn MFA を登録または通過する
6. `EMS ホーム` が開くことを確認する
7. `設定 > 端末情報` で登録状態を確認する

### HOSPITAL 利用者がやること

1. PC でログイン画面を開く
2. `ID / パスワード` を入力する
3. `端末登録` 画面が出たら、登録コードを入力する
4. 端末登録完了後に再ログインする
5. WebAuthn MFA を登録または通過する
6. `受入要請一覧` が開くことを確認する
7. `設定 > 端末情報` で登録状態を確認する

## 運用開始後の通常利用

### 通常ログイン

- `ID / パスワード` を入力して入る

### 5時間経過後

- `ID / パスワード + WebAuthn MFA` で再ログインする

## よくある誤解

### MFA を登録した = 端末登録完了

- 違う
- MFA は追加本人確認
- 端末登録完了かどうかは `設定 > 端末情報` で確認する

### display name がログインID

- 違う
- ログインに使うのは `ID / username`
- 画面に出る名称は `display name`

### 登録コードは毎回必要

- 違う
- 最初の端末登録時だけ必要

## 端末紛失時の新端末引継ぎ

### 全体像

1. 利用者が紛失に気づく
2. 所属責任者または `ADMIN` に連絡する
3. `ADMIN` がアカウントを停止する
4. 既存セッションを失効させる
5. 紛失端末を運用上の紛失扱いにする
6. 新端末または予備端末を決める
7. `ADMIN` が新端末向け登録コードを発行する
8. 利用者が新端末でログインする
9. 新端末で登録コードを入力する
10. 新端末で WebAuthn MFA を登録または通過する
11. `ADMIN` がアカウントを再開する

### 利用者がすぐ伝えるべきこと

- 名前
- ロール
- 端末名
- 最後に使った時刻
- 紛失場所の目安

### ADMIN がやること

1. 本人確認
2. アカウント停止
3. 必要なら一時パスワード再発行
4. 新端末の準備
5. 新しい登録コード発行
6. 再登録完了後にアカウント再開

### 新端末側でやること

1. `ID / パスワード` でログイン
2. `端末登録` 画面で登録コード入力
3. 再ログイン
4. WebAuthn MFA を登録または通過する
5. `設定 > 端末情報` で登録済み確認

## 確認ポイント

### 導入担当向け

- 利用者へ `ID` と `display name` の違いを説明したか
- 一時パスワードの有効期限を伝えたか
- 登録コードは最初だけだと説明したか
- WebAuthn MFA と端末登録は別の確認であると伝えたか

### ADMIN 向け

- 誤った所属へ端末を紐づけていないか
- 紛失時は先に停止しているか
- 再登録完了前に再開していないか

### 利用者向け

- 自分のログイン ID を把握しているか
- 端末名を把握しているか
- 紛失時の連絡先を知っているか
