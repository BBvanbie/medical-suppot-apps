# ネットワーク安全管理 Runbook

更新日: 2026-04-19

## 目的

- 搬送調整支援システムとして、`EMS`、`HOSPITAL`、`ADMIN` の接続元、禁止事項、障害時対応、責任分界を固定する。
- 医療情報システム安全管理ガイドライン上必要な `ネットワークに関する安全管理` を、このプロジェクトの現実的な運用前提に合わせて整理する。
- `構成図`、`接続点`、`外部接続条件`、`無線 LAN`、`FW / ACL`、`監視点`、`例外時の判断ルール` を 1 本の運用文書で追える状態にする。

## この文書の前提

- 本システムは当面 `搬送調整支援システム` を主体として扱う。
- 正式な診療録等の保存主体としての要件は、この文書では扱わない。
- 接続方法は、技術名よりも `安全に制御できること`、`責任分界が明確であること`、`現場で運用可能であること` を重視する。
- この文書は `導入組織側のネットワーク構成図` と `接続先一覧` の運用ルールを定める。実際の IP、回線名、機器名は導入組織側の管理資料で持つ。

## Guideline 的に必要なこと

- ネットワーク構成図、接続先、関係者一覧を最新化し続けること。
- 外部接続点には FW / ACL 等の境界防御を置き、接続元と接続先を制限すること。
- オープンネットワークを通る場合は TLS1.2 以上、可能なら TLS1.3 を前提にし、必要に応じて VPN やクライアント認証を組み合わせること。
- 無線 LAN は利便性だけで許可せず、許可端末、認証方式、アクセスポイント管理、電波到達範囲、ゲスト系との分離を決めること。
- 例外的な外部アクセス、私物端末利用、公衆無線 LAN 利用を安易に認めないこと。
- 障害時や紛失時に、ネットワーク遮断、端末失効、代替連絡へ切り替える手順を持つこと。

## 管理対象

- `EMS` 用モバイル端末とモバイル回線
- `HOSPITAL` 院内端末、院内 LAN、院内無線 LAN
- `ADMIN` 本部端末、本部 LAN、管理用回線
- インターネット接続点、FW、ルータ、VPN 装置、プロキシ、DNS、証明書
- 監視対象となる接続ログ、認証ログ、端末登録ログ、回線障害ログ

## 導入時に固定するもの

| 項目 | 何を固定するか | 保管先 |
|---|---|---|
| ネットワーク構成図 | 拠点、回線、FW、無線 LAN、外部接続点、監視点 | 導入組織側の文書管理場所 |
| 接続先一覧 | 本システム、通知連携、監視、backup、保守先 | 導入組織側の文書管理場所 |
| 責任者一覧 | 拠点ネットワーク担当、システム運用担当、保守窓口 | `medical-safety-responsibility-assignment-template.md` 実名版 |
| 例外遮断手順 | 回線遮断、端末失効、権限移管、電話連絡への切替 | 本 runbook と incident runbook |
| 監視点一覧 | 何を誰が見るか、異常時の連絡順 | 監視 runbook と導入組織側の監視台帳 |

## ロール別の接続方針

### EMS

- 利用端末は `公用端末のみ`
- 標準接続は `モバイル回線`
- 私物端末の利用は禁止
- 公衆 Wi-Fi の利用は禁止
- モバイル回線不通時は、システム利用継続ではなく `電話連絡へ切替` を正本にする

### HOSPITAL

- 利用端末は `院内の公用端末のみ`
- 接続元は `院内LANのみ`
- 院内無線 LAN を使う場合は `業務用 SSID` に限定し、ゲスト系や共用 SSID と分離する
- 院内でも `管理外ネットワーク`、`ゲストWi-Fi`、`持込端末` は禁止
- 同一拠点内の複数公用端末の登録は許可する
- 拠点外からの例外アクセスは認めない
- 病院ごとの送信元制限は `推奨` とする

### ADMIN

- 利用端末は `本部PCなどの公用端末のみ`
- 接続元は `本部拠点ネットワークのみ`
- 拠点外アクセスは認めない
- 同一拠点内の複数公用端末の登録は許可する
- 送信元制限は `原則必須` とする
- 管理画面到達前の境界で `FW / ACL / 送信元制限` のいずれかを実施する

## 共通禁止事項

- 私物端末からのアクセス
- 公衆 Wi-Fi からのアクセス
- 緊急時を理由にした例外アクセス
- 管理外ネットワークからの `HOSPITAL` / `ADMIN` 利用
- 端末故障時に旧端末を生かしたまま新端末を追加運用すること
- 導入組織が管理しないクラウドストレージ、メッセージング、ファイル転送経路への医療情報持出し
- 保守作業を理由にした無制限なリモート接続

## 通信保護の原則

- アプリ入口までの通信は暗号化する
- オープンネットワークを経由する通信は `TLS1.2 以上` を必須とし、可能な限り `TLS1.3` を使う
- 認証、端末登録、重要操作は監査・監視対象にする
- 接続元の制御は、`EMS` は端末登録と運用統制、`HOSPITAL` と `ADMIN` は拠点ネットワーク統制を主軸にする
- `ADMIN` は拠点ネットワーク統制に加えて送信元制限を原則必須とする
- `HOSPITAL` は院内LAN統制を主軸とし、送信元制限は推奨とする
- `VPN` は将来追加してもよいが、現時点では `EMS` の必須要件にしない
- `SSL-VPN` を使う場合は安易に許可せず、クライアント型、証明書利用、偽サーバ対策の有無を確認する

## 境界防御と論理分離

- 外部ネットワークとの接続点には `ステートフル FW または同等機能` を置く
- `ACL` で接続元、接続先、ポート、プロトコルを必要最小限へ絞る
- `HOSPITAL` / `ADMIN` の業務ネットワークは、ゲストネットワークや一般事務ネットワークと論理的に分離する
- メール、Web 閲覧、ファイル交換と本システム利用が混在する場合は、少なくとも接続経路と運用ルールを分ける
- 使用しないポート、保守用の一時開放、未使用機器は放置せず閉塞または撤去する

## 無線 LAN の扱い

- `EMS` は公衆無線 LAN を使用しない
- `HOSPITAL` が無線 LAN を使う場合は、業務用 SSID、強固な認証、許可端末の限定を前提にする
- `ADMIN` は原則として有線または管理下の無線 LAN を使い、共用 SSID を使わない
- 許可された者以外が利用できないよう、端末単位の制御、証明書、802.1X 等の組合せを検討する
- アクセスポイントの追加、移設、設定変更は台帳と構成図を同日更新する

## 外部接続と例外ルール

- `HOSPITAL` と `ADMIN` の拠点外アクセスは原則禁止とする
- やむを得ず保守接続を認める場合は、企画管理者承認、接続時間の限定、接続元の限定、作業記録、終了後の遮断確認を必須にする
- 利用者による恒常的な外部アクセスは、本プロジェクトでは前提にしない
- 将来 API / 電子カルテ連携を追加する場合は、接続先ごとに別紙で `接続経路 / 相互認証 / 暗号化 / 責任分界` を定義する

## 構成変更時の手順

1. 変更対象の回線、FW、ACL、無線 LAN、端末影響を整理する
2. 構成図と接続先一覧の更新有無を確認する
3. 企画管理者またはネットワーク責任者の承認を得る
4. 変更実施後に疎通、遮断、監視アラート、監査記録の反映を確認する
5. 変更記録を導入組織側の台帳へ残す

## 障害時の切替方針

### EMS

- モバイル回線不通時は電話連絡へ切り替える
- システム例外接続は認めない
- 端末故障時は予備の公用端末へ切り替え、必要な権限移管を行う

### HOSPITAL / ADMIN

- 拠点ネットワーク障害時は、まず拠点ネットワーク管理者が一次対応する
- 拠点外アクセスの例外は認めない
- 端末故障時は同一拠点内の別の公用端末へ切り替える
- 拠点全体が利用不能な場合は、代替運用または別拠点の公用端末への権限移管で対応する
- 不正アクセス疑いがある場合は、対象回線、端末、接続元を遮断してから原因調査へ移る

## 紛失 / 不正アクセス時の初動

1. 対象端末、拠点、回線、利用者を特定する
2. 必要に応じて端末失効、アカウント停止、送信元遮断を実施する
3. ログを保全し、影響範囲を切り分ける
4. 導入組織の連絡網に従って報告する
5. 代替運用へ切り替え、患者対応の継続性を優先する

## 責任分界

| 領域 | 主担当 | 補足 |
|---|---|---|
| `EMS` モバイル回線利用 | 現場運用責任者 / システム運用担当 | 不通時は電話運用へ切替 |
| `HOSPITAL` 院内LAN | 院内ネットワーク管理者 | ゲストWi-Fiや管理外ネットワークは禁止 |
| `HOSPITAL` 業務用無線 LAN | 院内ネットワーク管理者 | 業務用 SSID、認証方式、分離設定を維持する |
| `ADMIN` 本部ネットワーク | 本部ネットワーク管理者 | 可能なら送信元制限を行う |
| アプリ側の認証 / 監査 / 監視 | システム運用担当 / 開発保守担当 | ログイン失敗、権限逸脱、端末登録失敗等を監視 |
| 境界防御機器、FW、ACL | 拠点ネットワーク管理者 / 委託先 | 変更時は承認と記録が必要 |
| 障害時切替判断 | システム運用担当 / 現場責任者 | 例外アクセスは認めず、代替運用へ切替 |

## 監視対象

- ログイン失敗
- 権限逸脱
- 端末登録失敗
- 主要管理操作
- 異常なアクセス元の兆候
- FW / WAF / プロキシ等の遮断イベント
- 拠点ネットワーク障害、無線 LAN 障害、証明書期限切れ
- 保守接続の開始 / 終了記録

## 点検頻度

- 日次: 監視画面でネットワーク到達性、主要障害、異常アクセス兆候を確認する
- 月次: 構成図、接続先一覧、FW / ACL 変更履歴、証明書期限、無線 LAN 設定差分を確認する
- 半期: ネットワーク責任者、委託先、例外接続ルールを見直す

## 運用で記録すること

- 端末切替
- 権限移管
- 拠点ネットワーク障害
- モバイル回線不通による電話切替
- 不正アクセス疑い
- FW / ACL / 無線 LAN の設定変更
- 保守接続の承認、開始、終了、遮断確認
- 構成図と接続先一覧の更新履歴

## 記録保管場所の決め方

- 実記録の保管場所は `medical-safety-responsibility-assignment-template.md` 実名版の `記録保管場所` 表で固定する
- 構成図、接続先一覧、変更履歴、例外承認記録は同じ保管責任者の管理下に置く
- 監査時に 5 分以内で参照できない資料は `未固定` とみなし、是正対象にする
