# 端末登録ガイド

## 目的

- `EMS` と `HOSPITAL` は登録済み端末だけを使えるようにする。
- ふだんのログインは今までどおり `ID / パスワード` を使い、最初の 1 回だけ登録コードで端末を正式登録する。
- 運用担当、現場利用者、導入担当の全員が同じ説明で動けるようにする。

## このプロジェクトの現在の前提

- `EMS` は主に iPad やタブレットを使う
- `HOSPITAL` は主に院内 PC を使う
- 端末ごとに 6 桁 PIN を設定する
- PIN は 5 回失敗すると一時ロック対象になる
- 3 時間無操作で PIN 再開画面になる
- 8 時間経過で PIN では戻れず、`ID / パスワード` から再ログインする
- 紛失時は「端末だけ失効」ではなく「アカウント停止」を正本にする

## 誰が何をするか

- `ADMIN`
  - 登録コードを発行する
  - 必要時にアカウント停止 / 再開を行う
- 利用者
  - ログインする
  - 登録コードを入力する
  - 端末専用 PIN を設定する
- 導入担当
  - 端末名と配布先の整合を取る
  - 初回ログイン完了まで補助する

## 事前に準備するもの

- 利用者のログイン ID
- 一時パスワード
- 配布先端末
  - 例: `EMS-iPad-01`, `Hospital-PC-ER-01`
- 登録コード
- 問い合わせ先

## 管理者が先にやること

1. `ADMIN` でログインする
2. 端末管理画面を開く
3. 対象ユーザーまたは対象端末を確認する
4. `登録コード発行` を押す
5. コードを本人へ伝える

管理者の確認ポイント:

- コードは本人確認後に発行する
- 間違った端末名に紐づけない
- 使い終わったコードは再利用しない
- 誤送信や紛失の疑いがあれば、再発行で上書きする

## EMS 利用者の操作

例: iPad を `EMS` 用端末として登録する場合

1. iPad でログイン画面を開く
2. `ID / パスワード` を入力する
3. 初回または未登録端末なら `端末登録` 画面へ進む
4. 管理者から受け取った登録コードを入力する
5. 登録完了後はいったん再ログイン画面へ戻る
6. 次のログインで、この iPad 専用の 6 桁 PIN を設定する
7. `EMS ホーム` が開くことを確認する
8. 必要に応じて事案作成や病院検索も確認する
9. `設定 > 端末情報` で「登録済み端末」と `PIN: 設定済み` を確認する

## HOSPITAL 利用者の操作

例: PC を `HOSPITAL` 用端末として登録する場合

1. PC でログイン画面を開く
2. `ID / パスワード` を入力する
3. 初回または未登録端末なら `端末登録` 画面へ進む
4. 管理者から受け取った登録コードを入力する
5. 登録完了後はいったん再ログイン画面へ戻る
6. 次のログインで、この PC 専用の 6 桁 PIN を設定する
7. `受入要請一覧` が開くことを確認する
8. 必要に応じて `患者一覧`、`設定` も確認する
9. `設定 > 端末情報` で「登録済み端末」と `PIN: 設定済み` を確認する

## 登録後の通常利用

### 通常ログイン

- `ID / パスワード`

### 3 時間放置後

- 6 桁 PIN で再開

### 8 時間経過後

- `ID / パスワード` で完全再ログイン

## よくある見え方と意味

### ログイン直後に端末登録画面へ進む

- その端末はまだ正式登録されていない
- 登録コード入力が必要

### PIN 画面が出る

- 3 時間以上操作がなかった
- 同じ端末なら PIN で戻れる

### もう一度 `ID / パスワード` を求められる

- 8 時間以上経過している
- セッション期限のため完全再ログインが必要

### ログインできない

- パスワード相違
- 一時パスワード期限切れ
- アカウント停止中
- 登録端末条件を満たしていない
- ログイン失敗回数制限にかかっている

## パスワードを忘れたとき

1. 利用者が電話で `ADMIN` に連絡する
2. `ADMIN` が一時パスワードを発行する
3. 利用者は一時パスワードでログインする
4. ログイン直後に新しいパスワードへ変更する

補足:

- 一時パスワードの有効期限は 24 時間
- 一時パスワードのまま使い続けることはできない

## 端末を紛失したとき

1. 利用者がすぐに `ADMIN` へ連絡する
2. `ADMIN` がアカウントを停止する
3. 新しい端末または予備端末を決める
4. `ADMIN` が新しい登録コードを発行する
5. 利用者が新端末で再登録する
6. `ADMIN` がアカウントを再開する

## 導入担当が確認すべきこと

- 端末名と配布先が一致している
- 一時パスワードと登録コードの受け渡しが済んでいる
- PIN まで設定済みである
- `EMS` なら病院検索、`HOSPITAL` なら受入要請一覧まで確認済みである
- 紛失時の連絡先を利用者が把握している
