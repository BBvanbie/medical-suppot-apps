# 運用管理者向け手順書

## 目的

- `ADMIN` がこのシステムを日常運用するうえで、何を毎日確認し、いつ誰に介入するかを固定する。
- 現状実装されている認証 hardening、監視、端末登録、TRAINING 運用を前提に、迷わず辿れる運用導線を残す。

## 対象画面

- `ADMIN ホーム`
- `ADMIN / 監視`
- `ADMIN / ユーザー管理`
- `ADMIN / 端末管理`
- `ADMIN / 組織管理`
- `ADMIN / 設定`

## ADMIN が見るべき現在の主要機能

- ログイン失敗回数制限とロック
- 一時パスワード発行
- 強制パスワード変更導線
- 端末登録コード発行
- 3時間無操作後の PIN ロック
- 8時間経過後の完全再ログイン
- `ADMIN / 監視` の異常一覧
- `TRAINING` データの一括リセット

## 日次運用

### 朝の確認

1. `ADMIN / 監視` を開く
2. 次の 6 項目を確認する
   - `アプリ生存監視`
   - `DB 接続状態`
   - `ログイン失敗急増`
   - `重要 API 失敗`
   - `通知生成 / 配信失敗`
   - `バックアップ成否`
3. 赤や warning が出ている項目は、時刻と件数を記録する
4. 重大な異常があれば [incident-response-runbook.md](/C:/practice/medical-support-apps/docs/operations/incident-response-runbook.md) に従って初動に入る

### ログイン / 端末まわりの確認

1. ロック中アカウントが増えていないかを見る
2. 一時パスワード発行が必要な依頼を処理する
3. 端末登録待ちの現場があれば、登録コード発行まで進める
4. 紛失や故障の報告があれば、停止と再配布を優先する

### TRAINING 利用の確認

1. `TRAINING` 運用予定がある日かを確認する
2. 対象ユーザーの `currentMode` が意図どおりか確認する
3. デモ終了後に `LIVE` へ戻し忘れがないか確認する

## 週次運用

- 非アクティブユーザーの棚卸し
- 一時パスワード未消化の確認
- ログインロックの偏り確認
- 端末一覧と実配布状況の突合
- TRAINING データ残存状況の確認
- バックアップ 14 日保持が崩れていないかの確認

## 月次運用

- 所属、ロール、端末の棚卸し
- 紛失扱い端末の整理
- 退職 / 異動済みユーザーの停止漏れ確認
- 運用 runbook の更新要否確認
- 主要フローの軽い読み合わせ
  - ログイン
  - 端末登録
  - 受入要請一覧確認
  - 相談送受信
  - 監視確認

## ユーザー管理の標準操作

### 新規発行

1. 申請内容を確認する
2. ロール、所属、表示名を設定する
3. ユーザーを作成する
4. 一時パスワードを発行する
5. 必要なら端末登録コードも発行する
6. 利用者へ渡した内容を記録する

### ロック解除

1. 対象ユーザーを検索する
2. ロック理由を確認する
3. 本人確認が取れたら `ロック解除` を実行する
4. 再発防止のため、必要なら一時パスワードも発行する

### 一時パスワード発行

1. 電話などで本人確認を行う
2. 対象ユーザーで一時パスワードを発行する
3. 有効期限が 24 時間であることを伝える
4. ログイン後に強制変更されることを説明する

## 端末管理の標準操作

### 登録コード発行

1. 対象端末または対象ユーザーを特定する
2. `EMS` か `HOSPITAL` かを確認する
3. 端末名が運用上わかるものになっているか確認する
4. 登録コードを発行し、本人へ伝える
5. 必要なら登録完了まで待って画面確認を行う

### 紛失 / 故障対応

1. 端末名、利用者、最終利用時刻を確認する
2. アカウント停止を先に行う
3. セッション失効状態を確認する
4. 新端末または予備端末の準備後に再登録させる
5. 再開後に PIN 設定と主要画面表示を確認する

## 監視異常が出たときの判断基準

- `DB 接続異常`
  - ログイン不可や一覧取得不可に波及しやすいので最優先
- `重要 API 失敗`
  - 要請送信、相談、搬送判断に影響するため優先度高
- `通知失敗`
  - 代替運用を案内しつつ一次復旧を急ぐ
- `ログイン失敗急増`
  - 総当たり、運用ミス、配布情報不整合の可能性を切り分ける
- `バックアップ失敗`
  - 直ちに利用停止ではないが、その日のうちに再実行判断を行う

## 日常運用で最低限残す記録

- 一時パスワード発行日時
- ロック解除日時
- 端末登録コード発行日時
- 紛失 / 故障報告受付日時
- TRAINING 一括リセット実施日時
- 監視 warning / critical の対応開始と完了時刻
