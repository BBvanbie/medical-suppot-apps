# 医療情報安全管理 文書化に必要な入力情報一覧

更新日: 2026-04-19

## 目的

- ガイドライン対応文書を `仮データ` から `実データ` へ差し替えるときに、何を渡してもらえばよいかを 1 枚で整理する。
- 責任者割当、対外説明文書、委託先台帳、記録保管場所の差し替え漏れを防ぐ。

## 1. 導入組織情報

| 項目 | 例 |
|---|---|
| 導入組織名 | `○○総合病院`、`△△消防本部 第1隊` |
| 対象拠点 / 部門 | `本院 救急外来`、`第1方面隊` |
| 管理単位 | `病院単位`、`隊単位` |
| 文書管理番号 | `MSA-HOSP-001` |
| 初回承認日 / 最終更新日 | `2026-05-01` |

## 2. 責任者割当

| 必要情報 | 説明 |
|---|---|
| 管理責任者 | 氏名、役職、所属、連絡先、代行者 |
| 企画管理者 | 氏名、役職、所属、連絡先、代行者 |
| システム運用担当 | 氏名、役職、所属、連絡先、代行者 |
| 拠点ネットワーク担当 | 氏名、役職、所属、連絡先、代行者 |
| 現場責任者 | 氏名、役職、所属、連絡先、代行者 |
| 障害時連絡順 | 発見者から管理責任者までの連絡順 |

## 3. 記録保管場所

| 必要情報 | 説明 |
|---|---|
| 主保管場所 | 文書管理、共有フォルダ、チケット、GRC 台帳など |
| 副保管場所 | 主保管場所が使えない場合の参照先 |
| 閲覧権限 | 誰が見られるか |
| 保管責任者 | 誰が保管責任を持つか |
| 保管期間 / 見直し頻度 | 例: `3 年 / 月次` |

対象記録:

- 責任者割当実名版
- 連絡網 / 緊急連絡先
- ID 棚卸記録
- 本人確認記録
- 監査 / 点検記録
- Restore Drill / BCP 記録
- 資産台帳 / 教育記録
- 委託先契約 / SLA / 窓口一覧

## 4. 対外説明文書

| 必要情報 | 説明 |
|---|---|
| 対象文書 | `病院向け`、`EMS向け`、今後は `自治体向け` など |
| 導入先名 | 説明文書に対応する組織名 |
| 説明対象 | 誰に説明するか |
| 通常問い合わせ窓口 | メール、電話、担当部署 |
| 緊急連絡先 | 障害 / 事故時の優先連絡先 |
| 代替連絡手段 | 電話、FAX、既存連絡網など |
| 説明日 | 実施日 |
| 承認者 | 説明内容を承認する責任者 |

## 5. 委託先台帳

| 必要情報 | 説明 |
|---|---|
| 事業者名 | サービス名ではなく契約主体 |
| 委託カテゴリ | アプリ実行基盤、DB、backup、監視、通知連携など |
| 保存リージョン / 保存場所 | 例: `Japan / Tokyo` |
| SLA / 初報 | 初報時間、復旧目標など |
| 再委託有無 | あり / なし / 要確認 |
| 証跡 / 確認資料 | 契約、SLA、障害報告、監査回答、削除証跡など |
| 内部確認責任者 | 誰が確認責任を持つか |
| 病院向け開示可否 | 可 / 条件付き可 / 不可 |
| 状態 | 導入前 / 運用中 / 見直し中 |
| 契約終了条件 | データ返却、削除条件、ログ保全条件 |

## 6. 任意だがあると早い情報

- 問い合わせ受付時間
- 夜間 / 休日のエスカレーション順
- 監査回答窓口
- 既存の責任者一覧や連絡網ファイル
- 既存の契約一覧やベンダ管理台帳

## 7. 渡し方の最小単位

- 1. 導入組織情報
- 2. 責任者割当
- 3. 問い合わせ窓口
- 4. 委託先情報

この 4 まとまりがあれば、まず `病院向け` または `EMS向け` の実データ差し替えを進められる。
